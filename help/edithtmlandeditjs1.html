Before the page is loaded BuildHeader.js ensures that a user is logged in, if not the browser is


redirected to the login page.


This page is used to edit an existing image's information.


If the user is logged in, a request is sent to get the image and it's information. During this


a check is done to ensure the user is the owner or is the 'admin'. If it passes the page is


populated with the image and it's information.


An additional query is sent to get the groups that user belongs to so that the group options list


can be populated.


The user is then able to edit any information and then hit save to send the changes.


On save, the input is initially validated and processed client side before being sent to the server


where it once again validates that the user has permission to make changes (is the owner


or 'admin'). If it passes the validation the changes are saved.
